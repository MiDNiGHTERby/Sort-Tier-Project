# Sort-tier

**Краткое описание**  
Sort-tier — утилита для Windows, рекурсивно сканирующая каталог и группирующая файлы по месяцу создания (YYYY-MM) и по расширению.
Системные требования: python 3.8 и выше.

## Что изменено в последней версии
- Добавлены форматы графических редакторов и музыкальные форматы.
- Формат `.iso` теперь **перемещается** в целевую структуру (как обычные файлы).
- `exe` и `msi` по-прежнему **копируются** в целевую структуру; оригиналы остаются.
- При совпадении имени и формата в целевой папке сравниваются SHA256; при совпадении — автоматическая перезапись не происходит (файл пропускается).

## Быстрый старт
1. Скопируйте `sort-tier.py` в рабочую папку.
2. Запустите:
```bash
python sort-tier.py
```

## 📋 Полное описание функционала

### Основные возможности

#### 1. **Сортировка файлов**
- Сканирует директорию **рекурсивно** и находит файлы по выбранным категориям
- Группирует файлы по **дате** (папка вида `YYYY-MM-DD`)
- Затем распределяет их по **расширениям** (подпапки: `jpg`, `mp4`, `pdf` и т.д.)
- Структура результата: `<целевая_папка>/YYYY-MM-DD/<расширение>/файл`

#### 2. **Поддерживаемые категории файлов**
- **Изображения**: jpg, jpeg, png, gif, bmp, webp, tiff, tif, svg
- **Аудио**: mp3, m4a, flac, wav, aac, ogg, wma, alac
- **Видео**: mp4, mkv, mov, avi, wmv, flv, webm, m4v
- **Документы**: pdf, doc, docx, odt, txt, rtf, xls, xlsx, csv, ppt, pptx, md
- **Графика**: psd, psb, ai, eps, cdr, xcf, kra, sketch, ps, ora, pdn
- **Инсталляторы**: exe, msi
- **Архивы**: zip, rar, 7z, tar, gz, bz2, xz, torrent
- **Образы ISO**: iso

#### 3. **Два режима работы**
- **Переместить** — файлы переносятся в целевую папку (оригинальные удаляются)
- **Копировать** — файлы копируются, оригиналы остаются на месте

#### 4. **Умная обработка файлов**
- Обнаруживает **идентичные файлы** (по SHA256) и не дублирует их
- При конфликте имён генерирует уникальные имена (`file_1.jpg`, `file_2.jpg`)
- Исключает **системные файлы** (Windows атрибут)
- Автоматически обходит опасные папки:
  - `C:\Windows`
  - `C:\Program Files` и `C:\Program Files (x86)`
  - `C:\ProgramData`
  - Папки игровых клиентов (Epic Games, GOG, Steam, Origin, Riot Games)
- Поддерживает **dry-run** режим для предварительного просмотра всех операций

#### 5. **Восстановление после ошибок** (только при перемещении)
- Ведёт **подробный лог** всех операций в файле `sort-tier-<unix_ts>.log`
- Позволяет **полностью откатить изменения** и вернуть файлы обратно в исходные директории
- Создаёт отдельный лог восстановления `sort-tier-back-<unix_ts>.log` в исходной папке
- При копировании восстановление не предлагается

#### 6. **Интерфейс пользователя**
- **Графический UI** (при наличии зависимостей):
  - `InquirerPy` — красивые интерактивные диалоги выбора категорий и режимов
  - `tkinter` — системные диалоги выбора папок
  - `pyfiglet` — ASCII-арт баннер приложения
- **Fallback режим** — автоматический переход на текстовый ввод в консоли, если графические библиотеки недоступны
- **Прогрессбар** (tqdm) для отслеживания процесса обработки (только в консоли, отключён в fallback режиме)

#### 7. **Логирование операций**
- Все операции логируются в **UTF-8 кодировке** с временными метками
- Логи содержат: уровни ошибок, детали каждой операции, информацию для отката
- Помогают отследить и откатить любые изменения

## 🔄 Типичный workflow

1. **Выбор категорий** — отмечаешь какие типы файлов сортировать (изображения, видео и т.д.)
2. **Выбор режима** — переместить файлы или копировать
3. **Выбор папок** — указываешь исходную директорию для сканирования и целевую (или ту же самую)
4. **Подтверждение** — подтверждаешь выполнение (dry-run или реальное выполнение)
5. **Сортировка** — утилита обрабатывает файлы и организует их по структуре `YYYY-MM-DD/<расширение>/`
6. **Восстановление** (опционально) — если нужно откатить операцию перемещения, можешь восстановить файлы

## 📝 Логирование

- Логи **операций сортировки** сохраняются в: `<цел��вая_папка>/YYYY-MM-DD/sort-tier-<unix_timestamp>.log`
- Логи **восстановления** сохраняются в: `<исходная_папка>/sort-tier-back-<unix_timestamp>.log`
- Все логи содержат полную историю операций для отслеживания и анализа

## 💡 Особенности

- ✅ Безопасная работа — не трогает системные файлы и папки
- ✅ Умная дедупликация — не копирует одинаковые файлы
- ✅ История операций — всё логируется и может быть откачено
- ✅ Гибкий интерфейс — работает как с GUI, так и в консоли
- ✅ Эффективная обработка — использует прогрессбар и оптимизирует операции
